## ⚙️ FLUXO OPERACIONAL

### 🔎 FASE 1 — MAPEAMENTO DO APP
- Identificar:
  - rotas
  - páginas
  - dashboards
  - formulários
  - modais
  - componentes reutilizáveis
  - estados (loading, vazio, erro)
- Executar o app
- Navegar como usuário real
- Mapear fluxos principais

---

### 🧠 FASE 2 — AUDITORIA UX

Para cada tela:

- Nome da tela
- Objetivo do usuário
- Ação principal
- Problemas encontrados
- Lei de UX violada
- Impacto (clareza, decisão, velocidade, confiança)
- Severidade

---

## 🚨 CLASSIFICAÇÃO DE SEVERIDADE

- **CRÍTICO** → impede conclusão de tarefa  
- **ALTO** → gera confusão relevante  
- **MÉDIO** → reduz eficiência  
- **BAIXO** → melhoria estética  

---

## 🎯 FASE 3 — PRIORIZAÇÃO

Priorizar:

1. Bloqueios de fluxo
2. Alta carga cognitiva
3. Falta de hierarquia
4. CTA confuso
5. Inconsistência visual
6. Lentidão ou falta de feedback

---

## 🛠️ FASE 4 — AUTO-REFATORAÇÃO

### 🔧 Ações permitidas:
- Simplificar layouts
- Reduzir opções
- Agrupar informações
- Melhorar espaçamento
- Reforçar hierarquia visual
- Ajustar tipografia e contraste
- Padronizar componentes
- Melhorar CTAs
- Melhorar formulários
- Corrigir feedback (loading, erro, sucesso)
- Criar estados vazios úteis

### ⚠️ Regras:
- Não quebrar lógica de negócio
- Não remover funcionalidades sem justificativa
- Não alterar backend sem necessidade
- Não fazer mudanças apenas estéticas

---

## 🧪 FASE 5 — VALIDAÇÃO

Após mudanças:

- Rodar aplicação
- Testar no navegador
- Validar fluxos
- Confirmar melhoria de UX
